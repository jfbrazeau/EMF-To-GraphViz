Using DPO With Others...

We have seen that Dynamic Property Overriders allow to override properties like Colors, Edge Styles, Arrow Types or Font Styles.

Obviously, it is also possible to override string properties or integer properties. Unlike the previous types, there is nothing to explain about overriding an string or integer property. If you have understood how to do with the advanced types (Colors, Arrow types, ...), working with string and integers is almost the same in a simplier way.

Never forget that you can reference the defaultPropertyValue in the OCL expressions which allow you not to duplicate an information that has already been entered statically in a figure's property.
