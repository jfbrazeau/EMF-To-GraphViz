Basic concepts

Graphical description

The graphical description is the configuration resource that contains all the directives that emf2gv uses in the diagram generation process.

Emf2gv provides an eclipse based tree editor that helps to create graphical descriptions.

A graphical description basically contains ClassFigures, AttributeFigures and ReferenceFigures.

Note : it can also contain advanced items like RichAttributeFigures, RichReferenceFigures, Filters and DynamicPropertyOverriders.

The root node of a graphical description is called GVFigureDescription and accepts the following properties :

The graphical description accepts the following properties :

CategoryNameDescription
Appearance alignSameEClassesIf true, organizes the objects by type by applying a vertical or horizontal alignment (according to the chosen orientation)
orientationGraph orientation (top down or left to right)

Class figure

The ClassFigure contains the directives that will be used to render the diagram nodes.

It accepts the following properties :

CategoryNameDescription
Appearance bodyBackgroundColorFigure's body background color
headerBackgroundColorFigure's header background color
labelEAttributeThe EClass attribute to use for the figure's label (the label that appears in the figure's header)
labelStyleThe style of the class figure's label
Container nestedFiguresEReferencesThe EReference to use to retrieve the EObjects to draw as nested figures
Model eClassThe EClass that that this figure will represent on the graph

Attribute figure

The AttributeFigure contains the directives that will be used to render the EObject's attribute values.

That figure looks like an UML attribute in a UML Class diagram, which means taht it appears inside the EObject nodes, one above the other.

It accepts the following properties :

CategoryNameDescription
Appearance labelStyleThe style of the attribute figure's label
Model eAttributeThe EAttribute that this figure will represent on the graph

Reference figure

The ReferenceFigure contains the directives that will be used to render the EObject's references.

It accepts the following properties :

CategoryNameDescription
Appearance colorThe edge color
customSourceArrowThe custom source arrow (graphviz compliant arrow style, ignored if sourceArrowType is not set to 'custom')
customTargetArrowThe custom target arrow (graphviz compliant arrow style, ignored if targetArrowType is not set to 'custom')
minimumEdgeLengthMinimum edge length (same as Graphviz minlen property)
sourceArrowTypeThe source arrow type
styleThe edge style
targetArrowTypeThe target arrow type
Model eReferenceThe EReference that this figure will represent on the graph
